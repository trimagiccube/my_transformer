RoPE:把位置转进向量里

RoPE(Rotary Position Embedding,旋转位置编码)把 q/k 向量每两个数看成一根"指针", 按 token 的位置把它转一个角度。下面三个互动区,拖滑块就能看懂它在干什么、为什么有效。

1一对数 = 一根针,位置越大转得越多

取向量最前面的两个数 (v0, v1),画成一根从原点出发的针。 拖动「位置 pos」滑块:针会被转 θ = pos × 频率 度。注意针的长度始终不变,只是方向变。

原始向量 (v0,v1) 旋转后 (位置戳)
旋转角度 θ = pos×freq = 0°
原始针: (0.80, 0.60) 长度 1.00
旋转后: (0.80, 0.60) 长度 1.00
长度没变 → RoPE 不改变向量"大小",只改"方向"。方向就编码了"我在第几个位置"。

2288 个数 = 144 根针,转速不同

向量被两两配对成很多根针,它们转速不同:靠前的针转得快(像秒针,管"近距离"), 靠后的转得慢(像时针,管"远距离")。拖 pos,看它们一起转——快的转好几圈,慢的才动一点点。

就像钟表用秒/分/时针同时表示秒、分、时,RoPE 用不同转速的针同时编码近距离和远距离的位置关系。

3魔法:比较两个词时,只剩"位置差"

把同一对数,分别盖上「词A 在 posA」和「词B 在 posB」的位置戳。注意力靠 q·k 比较, 而点积主要看两根针的夹角。拖动两个位置滑块——夹角永远等于 (posB − posA)×频率,只跟"相隔多远"有关,跟绝对位置无关!

试试:把 A、B 一起往右拖(比如 A=0,B=3 → A=10,B=13),位置差不变 → 夹角和点积完全不变。这就是 RoPE 给注意力带来"相对位置感"的原理。

★三句话总结

1. 每两个数 = 一根针;RoPE 按 token 位置把针转一个角度(位置越后转越多),长度不变。

2. 转完比较两个词时,只剩"位置差"决定夹角 → 注意力自动知道两词相隔多远。

3. 144 根针转速不同(快针管近、慢针管远),像钟表的秒/分/时针配合。
